## ADRs (titles only)
- 0001-mcp-as-primary-interface.md: MCP server is the primary interface
- 0002-secrets-encrypted-in-db.md: Secrets stored encrypted in DB, not in .env
- 0003-invite-only-self-service-profiles.md: Invite-only self-service onboarding, one Owner Profile per invitee
- 0004-tiered-connections-login-dormant.md: Connections are tiered; login-based platform connections ship dormant
- 0005-multi-profile-serial-pipeline.md: Multi-profile pipeline: one tick iterates active profiles serially
- 0006-inference-via-shared-gpu-broker.md: Inference goes through a shared GPU broker, not direct Ollama calls
- 0007-ntfy-over-telegram-for-push-alerts.md: Push alerts are delivered via self-hosted ntfy, not Telegram

No ADR governs the stealth-driver choice itself — `docs/playwright-stealth-pilot.md` is the doc of record for that.
