# Requirements: Patchright Stealth Driver (narrowed pilot)

## Problem statement

`docs/playwright-stealth-pilot.md` frames the anti-bot pilot as a two-way bake-off between `rebrowser-patches` and Patchright for patching the CDP `Runtime.enable` leak that `playwright-extra` + `puppeteer-extra-plugin-stealth` cannot touch — but the driver swap the doc proposes was never wired: `PLAYWRIGHT_STEALTH_DRIVER` is read only by `scripts/verify-scrapers.ts`, which warns and silently falls back to `legacy` for any non-`legacy` value, and `packages/core/src/platforms/playwright/browser.ts` always launches the `playwright-extra` + stealth-plugin stack regardless of the env var. A 2026-07-18 independent benchmark (651 verdicts, 31 Cloudflare-protected targets, published 2026-05-13 / updated 2026-07-12) found `rebrowser-patches` now ties unpatched vanilla Playwright (24/29 OK) while Patchright is meaningfully ahead (25/29 OK) and ships as an actively maintained, drop-in package (`patchright-nodejs` v1.59.1, 2026-04-01). This resolves the bake-off in the doc's favor of Patchright and removes the reason to keep evaluating `rebrowser-patches`. The people affected: whoever operates the Depop and Poshmark scrapers (the two platforms that route through the shared stealth-browser layer) and needs their pass rate to reflect a real driver, not a mislabeled fallback; and future maintainers of `docs/playwright-stealth-pilot.md`, who currently have to read code to discover the env var does nothing.

## Users / stakeholders

- Fashion-monitor pipeline operator — runs scheduled scrapes against Depop and Poshmark and bears the consequences of a driver regression (empty results, bans).
- `scripts/verify-scrapers.ts` invoker — runs `npm run verify:scrapers` manually or in CI/cron to get posture + pass/fail data and needs the driver label to be truthful.
- Future contributor reading `docs/playwright-stealth-pilot.md` — needs the doc's gate ("passes live smoke on Depop/Poshmark") to be checkable against real driver-tagged data, not anecdote.
- `packages/core/src/platforms/poshmark/scraper.ts` and `packages/core/src/platforms/depop/playwright-fallback.ts` — the two existing consumers of `platforms/playwright/browser.ts` whose behavior must not silently change for the `legacy` default.

## Functional requirements

1. The system shall accept `PLAYWRIGHT_STEALTH_DRIVER=patchright` as a valid value and, when set, launch browsers/contexts in `packages/core/src/platforms/playwright/browser.ts` via the `patchright` Node.js package's Chromium launcher instead of `playwright-extra` + `puppeteer-extra-plugin-stealth`.
2. The system shall accept `PLAYWRIGHT_STEALTH_DRIVER=legacy` and, when set (or when the env var is unset), preserve the current `playwright-extra` + `puppeteer-extra-plugin-stealth` launch path unchanged.
3. The system shall no longer treat `rebrowser` as a recognized value of `PLAYWRIGHT_STEALTH_DRIVER` in any code path or doc — `rebrowser-patches` is dropped from consideration entirely, not merely deprioritized. This extends explicitly to `.claude/skills/fashion-monitor-change-control/SKILL.md`, `.claude/skills/fashion-monitor-research-frontier/SKILL.md`, `.claude/skills/resale-platforms-reference/SKILL.md`, and `docs/plans/stack-modernization.md`, none of which shall reference `rebrowser`/`rebrowser-patches` as a live, active, or recognized option once this feature ships.
4. The system shall warn (not throw) and fall back to `legacy` when `PLAYWRIGHT_STEALTH_DRIVER` is set to any value other than `patchright` or `legacy`, so a typo or a stale `rebrowser` value degrades safely instead of crashing a scheduled run.
5. The system shall expose the resolved driver as the return value of `resolveStealthDriver()` in `packages/core/src/platforms/playwright/browser.ts`, and as a `driver` field on both the `ScrapeReport` and `PostureCapture` interfaces in `scripts/verify-scrapers.ts`, so callers can label captures and logs with the driver actually in effect, rather than inferring it independently.
6. `ScrapeReport.driver` and `PostureCapture.driver` in `scripts/verify-scrapers.ts` shall be typed as the `StealthDriver` union type (`"patchright" | "legacy" | "n/a"`), not plain `string`, so a typo'd or mis-cased driver value cannot silently compile and fail downstream comparisons.
7. The system shall route both `launchStealthPersistentContext()` (consumed by `poshmark/scraper.ts`) and `launchStealthEphemeralBrowser()` (consumed by `depop/playwright-fallback.ts` and `verify-scrapers.ts`'s posture probe) through the same driver-selection logic, so Poshmark's persistent-profile flow and Depop's ephemeral fallback both honor `PLAYWRIGHT_STEALTH_DRIVER` identically.
8. The system shall preserve existing launch options (headless, `LAUNCH_ARGS`, `DEFAULT_USER_AGENT`, persistent-profile path keying) under the `patchright` driver so behavior differs only in which package launches the browser, not in launch configuration.
9. The system shall preserve `closeStealthPersistentContext()`, `closeStealthEphemeralBrowser()`, `closeAllStealthBrowsers()`, and `resetStealthStateForTests()` semantics for both drivers, so no Chromium process leaks regardless of which driver launched it. When multiple drivers have launched browsers within the same process (e.g., a driver-matrix run), these functions shall close every open resource across every driver exercised during the run — there is no caller-selectable mode to close only one driver's resources while leaving another's open.
10. `resolveStealthDriver()` shall read `PLAYWRIGHT_STEALTH_DRIVER` fresh on every call and shall not memoize or cache the resolved value, since the driver-matrix loop in `scripts/verify-scrapers.ts` depends on mutating the env var per iteration and re-resolving it each time to steer scraper factories.
11. `scripts/verify-scrapers.ts` shall replace its current `resolveDriver()` (which always returns `"legacy"` with a warning for any non-`legacy` request) with logic that actually launches the requested driver via the now-wired `browser.ts`, and shall label posture captures and scrape reports with the driver that actually ran.
12. `scripts/verify-scrapers.ts` shall support running the existing platform checks and posture captures once per driver (`legacy` and `patchright`) in a single invocation — a "driver matrix" — producing one labeled result row per platform per driver for Depop and Poshmark, so pass-rate comparison does not require two manual re-runs with the env var toggled by hand.
13. The driver-matrix row for Depop shall record the driver as a `"n/a"` (not applicable) sentinel value — not the configured/requested driver (`patchright` or `legacy`) — whenever Depop's impit-first HTTP path succeeds without invoking a Playwright/patchright browser, so the matrix does not reproduce the mislabeled-fallback problem this feature exists to fix.
14. The system shall continue to run eBay, Grailed, and Vestiaire's existing checks exactly once (not per-driver) in `verify-scrapers.ts`, since those platforms either have no anti-bot surface (eBay, Grailed are JSON APIs) or do not route through the stealth-browser driver for their primary path (Vestiaire's posture-only screenshot capture is unaffected by this driver swap).
15. The system shall add `patchright` as a declared dependency of `packages/core/package.json` without removing `playwright-extra` or `puppeteer-extra-plugin-stealth` from dependencies.
16. The system shall document `PLAYWRIGHT_STEALTH_DRIVER` in `.env.example` (it is currently undocumented there despite being read in code), including its two valid values and the default.
17. `docs/playwright-stealth-pilot.md` shall be updated to remove `rebrowser-patches` as an active candidate, record the 2026-07-18 benchmark result and its source characteristics (651 verdicts, 31 Cloudflare-protected targets, published 2026-05-13, updated 2026-07-12), and describe the driver swap as wired (not deferred).
18. The "do not remove yet" gate in `docs/playwright-stealth-pilot.md` shall be updated to reference Patchright by name and shall remain a hard requirement: `playwright-extra` + stealth-plugin code path shall remain present and reachable via `PLAYWRIGHT_STEALTH_DRIVER=legacy` until Patchright passes live smoke on both Depop and Poshmark.

## Non-functional requirements

- No change to headless/launch-arg configuration (`LAUNCH_ARGS`, `DEFAULT_USER_AGENT`) between drivers — differences in scrape outcome must be attributable to the driver's anti-detection behavior, not incidental config drift.
- No process/resource leak: every browser or context opened under either driver must be closed by the existing `close*` functions in the same run, verified by the existing `resetStealthStateForTests()`-based test pattern in `packages/core/tests/platforms/playwright-browser.test.ts`.
- No change to Poshmark's persistent-profile-path keying (`data/poshmark-profile` default) or Depop's HTTP-first / Playwright-fallback ordering — the driver swap only changes which package launches Chromium, not when Playwright is invoked.
- `verify-scrapers.ts`'s driver-matrix run must not affect the script's exit code beyond existing behavior — posture/matrix capture remains diagnostic only; only real scrape failures (per platform, independent of driver) fail the run, per the script's current contract.
- Rate/pacing behavior (`betweenQueriesMs`, existing scrape-discipline in `scrape-utils.ts`) is unchanged by this feature — no new request volume beyond one extra driver pass per matrix run for Depop and Poshmark.

## Constraints

- Must integrate with the existing shared module `packages/core/src/platforms/playwright/browser.ts` — both current consumers (`poshmark/scraper.ts` via `launchStealthPersistentContext`, `depop/playwright-fallback.ts` via `launchStealthEphemeralBrowser`) must keep working without their own call sites changing behavior when `PLAYWRIGHT_STEALTH_DRIVER` is unset or `legacy`.
- Must use `patchright` (the npm package, backed by `patchright-core`) — no Python Patchright, no `rebrowser-patches`, no Camoufox.
- Must not raise the pinned `playwright` version (`^1.52.0` in `packages/core/package.json`) as part of this work — that is a separate concern already tracked in `docs/plans/stack-modernization.md`.
- Must preserve the `Config`/`Platform` types and `PlatformScraper` interface used by `scripts/verify-scrapers.ts` — the driver matrix is additive instrumentation, not an API change to scraper factories (this refers specifically to `PlatformScraper` and the `create*Scraper` factory functions; it does not prohibit adding an optional `driverOverride` parameter to `browser.ts`'s internal `launchStealthEphemeralBrowser`/`launchStealthPersistentContext` functions, which are a separate, lower-level interface).
- Must not touch ScrapFly fallback or the cookie-harvest-then-HTTP pattern (Vestiaire's primary path, Depop's impit-first path) — those are unaffected by which package launches the stealth Chromium.
- Node >=24, pnpm workspaces, existing TypeScript/ESM (`.js` import extensions) conventions in `@fm/core` must be followed.
- `.env.example` additions must follow the existing format/section convention already used in that file.

## Out of scope

- Any further evaluation, benchmarking, or code path for `rebrowser-patches` — dropped from consideration per the 2026-07-18 research sweep, not deferred.
- Camoufox or any anti-detect-Firefox driver option.
- Removing `playwright-extra` + `puppeteer-extra-plugin-stealth` from the codebase or dependencies — the gate in `docs/playwright-stealth-pilot.md` (Patchright must pass live smoke on Depop and Poshmark first) is unchanged and this feature does not attempt to clear it.
- Wiring `login`-tier scraping (Poshmark/Depop/Vestiaire session-based) — out of scope per ADR-0004; login Connections remain dormant.
- Scheduled/cron automation of the driver-matrix run and writing driver-tagged outcomes into `integration_events` / `v_integration_daily` — this is the "Next" work item already named in `docs/playwright-stealth-pilot.md` and fashion-monitor-research-frontier F4; this feature only makes the matrix runnable and truthful on demand, not automated.
- Extending driver-matrix pass-rate comparison to Vestiaire or any platform beyond Depop and Poshmark — Vestiaire's existing posture-only capture is untouched, not expanded into a driver comparison.
- Changing `verify-scrapers.ts`'s exit-code contract, its non-driver-related output format, or eBay/Grailed's status-code-only capture path.
- UI, dashboard, or reporting surface for pass-rate data — this feature produces console output from `verify-scrapers.ts` only.
- Any change to `impit` TLS-impersonation logic, DOM selectors (`extract.ts`, `parse-rsc.ts`), or platform normalization.
- Defining a numeric/threshold closing criterion for the "passes live smoke" gate (how many runs, what pass rate counts as "passing") — this feature only makes driver-matrix data available; deciding when the gate is satisfied remains a human judgment call for a future change.

## Acceptance criteria

1. Running `verify:scrapers` with `PLAYWRIGHT_STEALTH_DRIVER=patchright` launches Chromium via the `patchright` package (verifiable by mocking `patchright` in a test analogous to `packages/core/tests/platforms/playwright-browser.test.ts` and asserting the patchright launcher, not `playwright-extra`, was called).
2. Running `verify:scrapers` with `PLAYWRIGHT_STEALTH_DRIVER=legacy`, or with the env var unset, produces identical launch calls to today's behavior: the `chromium.use`/`chromium.launch` assertions in the ephemeral-browser test and the `chromium.use`/`chromium.launchPersistentContext` assertions in the persistent-context test, both in `packages/core/tests/platforms/playwright-browser.test.ts`, pass unmodified.
3. Setting `PLAYWRIGHT_STEALTH_DRIVER=rebrowser` (or any other unrecognized value) produces a console warning and runs under `legacy` — it does not throw, does not silently label the capture as if `patchright` or `rebrowser` ran, and no code path references `rebrowser-patches` as a live option.
4. A single `verify:scrapers` invocation produces distinct, driver-labeled report rows for Depop and Poshmark for both `legacy` and `patchright` — each row carries `driver`, `statusCode`, and `screenshotPath` (or `error`) independent of the other driver's row for the same platform. The `driver` value is the literal `StealthDriver` string (`"patchright"` or `"legacy"`), except for Depop, where it is the `"n/a"` (not applicable) sentinel when Depop's impit-first HTTP path succeeded without invoking a browser.
5. eBay, Grailed, and Vestiaire rows in `verify:scrapers` output appear exactly once per run (not duplicated per driver).
6. `packages/core/package.json` lists `patchright` as a dependency; `playwright-extra` and `puppeteer-extra-plugin-stealth` remain listed unchanged.
7. `.env.example` contains a `PLAYWRIGHT_STEALTH_DRIVER` entry documenting `patchright|legacy` and the default (`legacy`).
8. `docs/playwright-stealth-pilot.md` no longer lists `rebrowser-patches` as an open candidate, cites the 2026-07-18 benchmark (651 verdicts / 31 targets / 2026-05-13 published / 2026-07-12 updated, 25/29 OK for Patchright vs 24/29 OK tie for rebrowser-patches/vanilla), and states the driver swap is wired rather than deferred.
9. `docs/playwright-stealth-pilot.md`'s benchmark citation includes its actual source — https://ianlpaterson.com/blog/anti-detect-browser-benchmark-patchright-nodriver-curl-cffi/ (published 2026-05-13, updated 2026-07-12) — not just the repeated statistic with no source pointer.
10. `docs/playwright-stealth-pilot.md`'s "do not remove yet" gate still requires `playwright-extra` + stealth-plugin to remain until Patchright passes live smoke on Depop and Poshmark — the gate is retargeted to Patchright, not weakened or removed.
11. Existing `packages/core/tests/platforms/playwright-browser.test.ts` and any new patchright-path test both pass in the same `vitest` run, and `resetStealthStateForTests()` correctly clears state between driver-specific test cases.
12. No orphaned Chromium process remains after a `verify:scrapers` run under either single-driver mode or matrix mode (verified via the existing `close*` call chain covering every browser/context opened during the run).
