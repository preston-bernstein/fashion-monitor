## CLAUDE.md
# fashion-monitor — execution routing

Cross-cutting home-lab conventions live in `home-infra/CONVENTIONS.md`. This file holds only
what's fashion-monitor-specific.

**Deploy target is the NAS, not this Mac checkout.** Images are built here and shipped:

```bash
make sync   # compose file, static config, grafana provisioning -> NAS
make push   # docker save fashion-monitor/cli fashion-monitor/mcp-server | ssh ... docker load
make deploy # sync + push, then docker compose up -d on the NAS
```

- Deployed/running copy: NAS Docker Compose stack (Makefile NAS_HOST/NAS_PATH).
- Live data: NAS-mounted SQLite DB + data/poshmark-profile (persistent Playwright cookies) —
  never present on this Mac checkout.
- Live scrapes are opt-in only (`pnpm run verify:scrapers`, `pnpm run test:live`) — default
  dev/test loop runs entirely against fixtures. Don't run live scrapers from this Mac clone
  casually; they hit real marketplaces.

Agent policy: no AI attribution in commits or code (.cursor/rules/no-agent-attribution.mdc).

## CONTEXT.md
Fashion Monitor domain glossary — Monitor, Taste, User, Invite, Profile, Role, Score
(YES/MAYBE/NO), Interface hierarchy (MCP server primary for owner, web app primary for
invited users, CLI for pipeline execution), Query Override, Secret, Connection (per-profile
per-platform link; three kinds: API-key/none/login; login-based platforms like
Poshmark/Depop/Vestiaire store session cookies, violate ToS, ban risk borne by user, off by
default; anonymous public scraping is the fallback when not connected), Pipeline (scrapes,
dedups, prefilters, scores, alerts on schedule), Scoring Dimensions (aesthetic/quality/value),
Feedback (seed + dashboard-sourced few-shot examples).

## Key live-investigation findings (feed into spec, do not re-derive)
- Depop's search page no longer embeds product data server-side at all (RSC-marker parsing
  is permanently dead, not just renamed).
- Real internal API found via live patchright network trace: `GET
  https://www.depop.com/presentation/api/v1/search/products/?what=<query>&limit=24&country=us
  &currency=USD&from=in_country_search&include_like_count=true`.
- That endpoint returned a genuine Cloudflare-fronted 403 (server: cloudflare, cf-ray present)
  on its very first call from the investigation's sandboxed environment — confirms real
  hardening, but the sandbox's egress IP is not the real NAS deploy IP, so this is not
  proof the primary HTTP path is doomed in production.
- A `sec-ch-ua: "HeadlessChrome"` header leak was observed on patchright's own request during
  the trace — a concrete, independently fixable browser-fingerprint bug.
- Success-response JSON shape was never observed live (every call was blocked) — fixtures and
  normalizer are provisional until the one opt-in live-verification step.
