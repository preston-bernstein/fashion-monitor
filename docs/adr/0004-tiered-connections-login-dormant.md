# Connections are tiered; login-based platform connections ship dormant

A **Connection** is a per-profile, per-platform link layering test + status + risk acknowledgment on top of `Secret`s. The six platforms are not uniform, so Connections are tiered by what the platform actually supports: **API-key** (eBay developer keys — sanctioned), **none** (Grailed — public Algolia search, no account to connect, shown as "automatic"), and **login** (Poshmark/Depop/Vestiaire — store the user's session and scrape *as them*). Login connections violate those platforms' ToS and carry an unappealable account-ban risk borne by the user, so they ship **dormant**: off by default, gated behind per-platform risk acknowledgment, and only enabled after (1) a per-platform ToS research pass produces honest risk copy and (2) a measured anonymous-vs-logged-in comparison proves a real result lift. When a platform is not connected, the pipeline falls back to anonymous public scraping, so the monitor works regardless. We rejected uniform login-for-all (contradicts the code and oversells) and API-only (would forbid a capability the Owner explicitly wants available). A Connection's Test writes an `integration_event` with `operation='test'` so manual tests share one uptime timeline with pipeline runs.
