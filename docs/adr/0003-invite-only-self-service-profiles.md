# Invite-only self-service onboarding, one Owner Profile per invitee

Fashion Monitor moves from admin-provisioned users to self-service onboarding, but **not** public registration. The system Owner generates a one-time **Invite** (a link); redeeming it creates the User, creates a fresh Profile, and makes that User the Profile's Owner. Each invitee therefore runs their own Taste, Monitors, Connections, and alert destination, fully isolated by `profile_id`. This breaks the earlier "one User maps to one Profile in v1" axiom (now M:N via `memberships`) and is hard to reverse once real second-tenant data exists. We chose invite-only over open signup to stay out of public-SaaS territory (no abuse moderation, email-verification, or billing) while still letting a trusted household member connect and manage their own service; over admin-provisioning because the whole point is that users connect their own accounts. Password reset reuses the same one-time-link machinery (no email infra). Profile deletion is a self-serve Owner action that cascades all `profile_id`-scoped rows, secrets, and memberships.
